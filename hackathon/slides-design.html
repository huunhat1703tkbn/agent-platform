Kanban AI-powered
Use Case 01
Quản lý → Assign đúng người cho task
Manager làm việc với nhiều team — rất khó biết ai đang làm gì, ai giỏi gì, ai còn capacity. Thay vì lướt nhiều trang, hỏi từng người hay tra Excel trước khi assign.
1
Manager nhắn: "List Infrastructure tasks that need to work on…"
2
Agent tìm task open → match skill tag → check workload & capacity
3
Hiện confirmation card — user chọn người, click Assign
HITL — Agent không tự assign
Use Case 02
PM / SM → Phát hiện task trùng lặp
Khi tạo task mới, PM thường không biết task đó đã có chưa, liên quan đến phần nào — dễ gây trùng lắp hoặc dev thiếu context để triển khai.
1
User tạo task mới trên Planner
2
Agent vector search tìm task tương tự → phân loại likely-dup / maybe-dup / no-match
3
Hiện card 3 lựa chọn: Link ticket cũ · Xóa · Ignore
No-match → tạo ngay Agent chỉ cảnh báo

2 Pattern chính

Chatflow vs Workflow

Hiểu rõ hai kiểu này trước khi thiết kế agent
Chatflow
Tương tác hội thoại theo lượt — user hỏi, agent trả lời ngay trong cùng một request.
→ Không có trạng thái lưu giữa các bước
→ Kết quả trả về ngay, không chờ duyệt
→ Dùng tool calls: search, query, summarize
Demo: Luồng 1 & 2 — tìm task, tìm member, đề xuất assignee
Workflow
Luồng có trạng thái persistent — suspend chờ người duyệt, resume sau khi được xác nhận.
→ Trạng thái lưu DB — đóng tab không mất
→ Bắt buộc có HITL trước mọi write
→ Card xuất hiện trong inbox, tự expire
Demo: Luồng 3 & 4 — assign by skill, dedup on create

Generator tạo đầy đủ scaffold — các đội chỉ cần điền nghiệp vụ

~2 hr
Thêm HTTP route + web screen + write tool với HITL confirmation.

Các bước fast path
1
pnpm gen module
Nhập tên module (kebab-case), chọn tier, gen web companion → scaffold sẵn trong packages/<name>/
2
pnpm --filter @seta/<name> db:generate
Viết Drizzle schema → gen migration SQL tự động

pnpm db:migrate
Chạy migration lên Postgres dev
4
reg.module({ agentTools })
Đăng ký tool vào agent runtime — tool xuất hiện trong chat ngay
5
pnpm dev
